A practical student-built platform combining threat detection, identity security, cloud posture checks, network monitoring, Zero Trust scoring, AI-assisted explanations and operator-confirmed cloud remediation.

Overview

The platform connects to AWS through boto3 and runs Streamlit inside a native Windows desktop window. Operators connect from the AWS page using an access key or existing AWS profile, then run individual scanners and download evidence without maintaining an external web server.

Main modules

  • Threat Detection: synthetic supervised-learning path plus real CloudTrail anomaly detection, brute-force detection and insider/external labeling.
  • IAM Security Audit: broad-policy identification, shadow-admin escalation checks and user-to-policy relationship mapping.
  • S3 Exposure Scanner: public-access configuration and ACL review, with a guarded button to enable all four bucket-level Block Public Access controls.
  • Security Group Scanner: dangerous ports exposed to the internet, with exact-rule metadata and a guarded button that removes only the selected public source.
  • VPC Flow Log Analyzer: suspicious network patterns such as repeated rejected connections.
  • Zero Trust Engine: identity trust scoring from permissions, MFA and behavioral findings.
  • Incident Reporting: consolidated findings with an AI-generated executive explanation and cloud-change audit trail.

IAM relationship graph

Blue nodes represent IAM identities, green nodes represent lower-risk policies, and edges show which policies are attached to each identity. This helps make broad access visible at a glance.

Security decisions

  • Credentials are entered in the local AWS page. Validated keys can optionally be protected by Windows Credential Manager; they are never placed in reports or source code.
  • Scanners use the included read-only IAM policy. The two fix buttons remain disabled by AWS unless an administrator separately attaches the narrow remediation policy.
  • The project is intended only for accounts the operator owns or is authorized to assess.

Install and connect

  1. Download the Windows ZIP and extract it completely. Do not run the installer from inside the ZIP preview.
  2. Double-click Install_RC_CloudSec.bat. Python 3.11 or 3.12 is required; the installer creates a private environment and an RC desktop shortcut.
  3. Open RC AI Cloud Security, select the AWS tab, choose a Region and connect a dedicated IAM identity.
  4. An AKIA key uses Access Key + Secret Key with no session token. An ASIA temporary credential requires Access Key + Secret Key + Session Token from the same session.
  5. Attach only RCCloudSecReadOnlyPolicy.json for scanning. Attach the optional remediation policy only after the AWS owner approves cloud changes.

Honest limitations

The supervised classifier uses synthetic login data, so its real-world accuracy is not assumed. Real CloudTrail activity is handled through anomaly detection and rules. Continuous verification runs only while the local application is open, and the platform is not presented as a production managed security service. One-button fixes are intentionally limited to selected S3 Block Public Access and Security Group ingress findings; IAM changes require manual review to avoid lockout.
